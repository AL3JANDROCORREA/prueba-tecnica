Agile Testing

Principios

  • Integración continua
  • Entrega continua
  • Elaborar menos documentación
  • Responder con velocidad a la retroalimentación
  • Proporcionar retroalimentación constante

Cuadrantes

  • Pruebas de cara al negocio

    • Pruebas funcionales y pruebas UAT
    • Enfoque: Pruebas manuales y críticas al producto

  • Pruebas de cara a la tecnología

    • Pruebas automatizadas y con herramientas

  • Pruebas soportando la programación o soporte del equipo

    • Pruebas unitarias y de componentes
    • Pruebas funcionales con prototipos y simuladores

  • Pruebas de críticas al producto

    • Pruebas con carácter positivo para sugerir mejoras
    • Simulan un ambiente real
    • Frecuentemente realizadas por usuarios finales (UAT)

Recomendaciones

  • Automatizar las pruebas de software
  • Los testers deben revisar las pruebas unitarias
  • Incluye a analistas de pruebas en el equipo de desarrollo
  • Realizar las pruebas en el proyecto al inicio, en cada sprint y constantemente

TDD

  • **RED:** Crear un test que falle
  • **GREEN:** Hacer que el test pase
  • **REFACTOR:** Mejorar el código

ATDD

  • **Seleccionar la historia de usuario**
  • **Escribir la prueba de aceptación**
  • **Implementar la historia de usuario - Automatizar las pruebas**
  • **Implementar el código**
  • **Realizar pequeños cambios/refactorización**

BDD

  • **Escribir un escenario**
  • **Ejecutar el escenario que falla**
  • **Escribir la prueba que corresponde a las especificaciones del escenario**
  • **Escribir el código más simple para pasar la prueba y el escenario**
  • **Refactorizar para eliminar la duplicación**

Testing exploratorio

  • **Exploración del producto**
  • **Diseño de pruebas**
  • **Ejecución de pruebas**
  • **Heurística**
  • **Resultados revisables**
